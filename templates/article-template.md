# 公众号 / 头条号排版模板

## 文件命名规范

`YYYY-MM-DD-标题.md`

例：`2026-06-03-蓝科高新收购中国空分.md`

---

## 标准结构

### 1. 正文前导（开场钩子）
```
## [一句总结性小标题或悬念]

[钩子段落 50-80 字]

[段落间隔]

[第二钩子段落或场景交代]
```

### 2. 主体论证（3-5 个小节）
```
### [小标题 1：现象 / 数据拆解]

[论证段落，100-200 字]

### [小标题 2：深层逻辑]

[论证段落]

...
```

### 3. 收尾（反思 + 开放问题）
```
---

[收尾段落，告诉读者"你怎么看"]

---

> **资料来源**
> 1. [来源名](URL)
> 2. [来源名](URL)
> 3. [来源名](URL)
>
> 注：[是否转述自公告，是否需自查等说明]
```

## 排版细则

### 字体强调
- 金句用 **加粗**
- 重点词汇用 **加粗**
- 避免<u>下划线</u>、~~删除线~~（公众号不支持）

### 标题层级
- `##` 大标题（文章名）
- `###` 小标题（段落名）
- 不用 `####` 或更深的层级

### 段落长度
- 单段最多 3 个自然段（约 200 字）
- 段间留一行空白（markdown 里是双换行）

### 列表用法
- 用 `- ` 做项目符号
- 避免`1. 2. 3.` 式编号（除非有递进关系）

### 数字格式
- 保留原始数据格式（1.31 亿，6481 万）
- 数据后跟单位（亿元、万元、%）
- 无来源数据标注"未披露"或"待核实"

### 引用格式
```
> 这是一个引用
>
> 多行引用用 > 符号
```

### 资料来源块

```
---

> **资料来源**
> 1. 新浪财经《标题》— https://example.com
> 2. 证券时报《标题》— https://example.com
> 3. 上交所公告 — https://example.com
>
> 注：[必要时说明是否核实原始文件]
```

## Markdown to mdnice 排版要点

1. 复制上述 markdown → [mdnice](https://editor.mdnice.com)
2. 选择主题（建议"技术简书蓝""姫色""简约"之一）
3. 点「复制」按钮 → 粘到公众号草稿箱
4. 标题、摘要、配图由公众号后台填（markdown 里不包含）
