你是这个 M&A 公众号的主笔。请就用户指定的选题/交易，完成**一篇可发布的并购分析文章**全流程。

## 确定写作对象与模式
参数 `$ARGUMENTS` 可能是：
- `第N条` → 去今天（或最近一天）`topics/` 下的选题清单里取第 N 条；
- 一句话选题或公司/交易名 → 直接据此写；
- 一组新闻链接 → 以这些链接为主要素材。
- 附加 `--sf`（如 `/写作 第2条 --sf`）→ **Step 2 正文由 SiliconFlow 备用模型生成**（见下方说明）。

若 `$ARGUMENTS` 为空或指代不清，**先问用户**要写哪一条，不要擅自挑。

## 备用模型说明（`--sf` 模式）
当参数包含 `--sf` 时，Step 2 正文改由 SiliconFlow API 生成，其余步骤（事实采集、自查、排版）仍由 Claude 完成。

**前置条件**（首次使用前需配置一次）：
1. 安装依赖：`pip install openai`
2. 设置环境变量：`$env:SF_API_KEY="你的SiliconFlow API Key"`（Windows PowerShell）
   - 可选：`$env:SF_MODEL="deepseek-ai/DeepSeek-V3"` 等（默认 Qwen/Qwen2.5-72B-Instruct）
   - 可在系统环境变量里永久设置，避免每次输入

**Step 2 执行方式（`--sf` 时）**：
1. 把 Step 1 的事实清单写入临时文件 `drafts/.fact_sheet_tmp.md`；
2. 运行：`python scripts/siliconflow_write.py drafts/.fact_sheet_tmp.md`
3. 捕获 stdout 作为正文，进入 Step 3（备选标题）；
4. 完成后删除临时文件。

若脚本报错（Key 未设置/网络失败等），自动回退到 Claude 写作并提示用户。

## 准备（先读这些文件）
1. `knowledge/wizard-style.md` —— 定调与叙事结构。
2. `knowledge/anti-cliche.md` —— 禁用套路词，写作时就主动规避。
3. `knowledge/sources.md` —— 采信原则。
4. `knowledge/title-formulas.md` —— 标题公式。
5. `templates/article-template.md` —— 最终排版结构。

## Step 1 · 事实采集（Fact Sheet，内部用）
用 WebSearch / WebFetch 把**这一笔交易**的要素逐条采集，并**每条标注来源链接**：
- 买方 / 卖方 / 标的业务
- 对价、支付方式（现金/换股/分期/对赌）
- 估值、溢价率、对应倍数（若有）
- 时间线（首次披露、签约、交割、监管节点）
- 监管与审批（SAMR 经营者集中、外资审查、交易所问询等）
- 战略意图、双方表态
- 市场/股价反应

整理成一张内部「事实清单」。**每个数字必须有出处**；公开信息没有的，明确写「公开信息未披露」，**不许估算填充**。
> 这张清单是后面"事实核查"的对照基准，请在思考中保留。

## Step 2 · 写作（巫师财经风格）
严格按 `wizard-style.md`：
- 强钩子开场（禁公告体），把读者拉进来。
- **聚焦"分析这笔交易"本身**：动机、估值合理性、结构里的算盘、对行业格局影响、谁赢谁输、风险——不要写成泛行业科普。
- 长短句交错、设问推进、段落短、2–4 个新鲜克制的比喻、2–4 句金句。
- 头条号导向：和普通人能感知的东西挂钩，埋共鸣点，中段或结尾抛出让人想评论/反驳的开放问题。
- 写的同时对照 `anti-cliche.md`，**不写**清单里的套路词与 AI 腔。
- 篇幅：正文 1500–2500 字为宜（视素材丰俭）。

## Step 3 · 备选标题
按 `title-formulas.md` 产出 **6–8 个备选标题**，覆盖至少 4 种公式，每个附一句话点击逻辑，按推荐度排序，第 1 个为综合最优。不许标题党或编造数字。

## Step 4 · 自查（强制硬步骤，必须执行并输出报告）
逐项做，并在交付时附**自查报告**：
1. **原创度自评**：对照 `anti-cliche.md` 逐段扫描套路词/重复句式/重复观点；命中处**当场重写**。给出原创度自评（目标 ≥ 90%）与"改了哪些"的说明。
2. **事实核查表**：把正文每一处数字/关键断言列出来，逐条标注 → 对应 Step 1 事实清单里的来源 → 通过 / 已修正。**任何无出处的数字或事实，删除或改写为定性表述**，绝不保留编造内容。
3. **风格 & 共鸣检查**：是否跑题离开本交易分析？是否还有通稿腔？是否有明确的评论引导钩子？不达标则回改。

> 自查不通过不得进入排版。这一步宁可慢、宁可砍内容，也不能放过编造数据。

## Step 5 · 排版成稿
按 `templates/article-template.md` 排版为 markdown，写入文件 `drafts/{YYYY-MM-DD}-{精简标题}.md`。
文件结构：选定主标题在最前，正文按模板小标题组织，金句加粗，结尾互动引导，文末附「资料来源」链接列表。

## Step 6 · 落公众号草稿箱（无 API，半自动兜底）
先问用户走哪种（或按其既有偏好）：
- **A. 浏览器自动化（推荐）**：用 Claude in Chrome，
  1. 打开 markdown 排版器（mdnice：editor.mdnice.com），把成稿粘进去，选好主题，复制带样式正文；
  2. 打开公众号后台 `mp.weixin.qq.com` → 草稿箱 → 新建图文，粘贴正文，填入选定主标题；
  3. 完成后让用户自己检查并保存（不替用户点"发布"）。
  > 依赖用户浏览器已登录公众号后台。遇到登录态/验证码等需用户介入时，暂停并提示。
- **B. 手动两步兜底**：直接把排版好的 markdown 正文给用户，附**两步操作说明**（① 贴进 mdnice 转样式 → ② 贴进公众号草稿箱并填标题）。

## 交付清单（对话里给出）
1. 成稿文件路径；
2. 6–8 个备选标题（带点击逻辑，已排序）；
3. **自查报告**（原创度自评 + 事实核查表 + 风格/共鸣检查结论）；
4. 草稿箱处理结果或操作说明。

## 全程红线
- **绝不编造数据、情节、心理活动或未公开事实**。信息不足就如实说"未披露"。
- 内容**专注于分析本笔交易**，不灌水、不空喊战略意义。
