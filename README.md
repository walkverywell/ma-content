# 并购公众号自动化工作流

## 快速开始

### 命令 1：每日选题
```
/选题
```
执行：实时搜索过去 48 小时中国公司相关并购新闻 → 去重 → 打分 → 生成 6-10 条选题清单 → 存到 `topics/YYYY-MM-DD.md` → 通知你

**使用时机**：每天早上自动跑，或手动触发更新选题

---

### 命令 2：风格化写作
```
/写作 第1条
```
或指定选题：
```
/写作 蓝科高新 1.31亿收购中国空分
```

执行流程：
1. **事实采集**：WebSearch + WebFetch 回一手公告核实每个数字（带来源）
2. **风格化成文**：按巫师财经风格写（钩子+悬念+数据对话+利益拆解）
3. **生成备选标题**：6-8 个标题，覆盖悬念/数据/共鸣/讽刺型
4. **强制自查**：
   - 原创度自评（扫描 anti-cliche.md 里的套路词，≥90% 原创）
   - 事实核查（每个数字回链来源，无出处改为"未披露"）
   - 风格检查（有开放式问题？有共鸣钩子？）
5. **排版**：按 article-template.md 格式成 markdown
6. **推公众号**：
   - A 方案：用 Claude in Chrome 自动化——mdnice 排版 → 公众号草稿箱
   - B 方案：生成"一键复制"材料 + 两步手动说明

成稿存到 `drafts/YYYY-MM-DD-标题.md`

## 反编造护栏

**硬规则** — 写作前必读，违反任何一条都会被自查卡住：

1. ✓ 每个数字都必须有来源链接或标注"未披露"
2. ✓ 选题清单里的数字需回一手公告 / 财报二次核实
3. ✓ 任何编造的数据一律删除或改写为定性表述
4. ✓ 不确定的表述不要强行解释，标注"待核实"
5. ✓ 正文里禁用 anti-cliche.md 的套路词

## 常见问题

### Q1：选题怎么选？
看 `knowledge/topic-scope.md` 的"选题价值判断维度"。简版：热度 + 独特性 + 可分析空间 + 共鸣度。

### Q2：标题怎么选？
看 `knowledge/title-formulas.md`。建议首选"悬念反差 + 共鸣型"混合，点击+评论都高。

### Q3：如果数据查不到怎么办？
**不能编造**。改成"未披露"或"待核实"，或删除这个断言，改写为定性表述。

### Q4：能用外部 AI 写吗？
默认用 Claude Code（Opus）写。若要接入你已有的模型，可在 `/写作` 里加可选分支（需提供调用方式）。

### Q5：公众号草稿箱接口怎么用？
暂时没有 API（你的号还没认证）。目前用 B 方案（手动两步）或 A 方案（浏览器自动化）。待你拿到认证服务号 + AppID + IP 白名单后，可升级为全自动 API 推送。

```

  ↓

  ↓
```

## 使用建议

1. **前三篇试稿**：先手动全流程跑一遍，熟悉每一步
2. **逐步自动化**：先开启每日选题定时，再升级其他环节
3. **定期微调**：每周看看 anti-cliche.md 和 title-formulas.md 需不需要补充新素材
4. **保持原创**：宁可不发，也别为了出稿量而降低标准
